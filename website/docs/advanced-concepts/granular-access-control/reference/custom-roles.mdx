To secure applications and provide access based on your business needs, Appsmith allows you to create your own custom roles. This page showcases different scenarios and examples for creating custom roles to manage granular access effectively.

## Restrict page access

Controlling access to specific pages within your Appsmith application is vital for maintaining security and privacy. This page provides information on how to restrict page access to designated users or user groups, enabling you to enforce access control effectively.

### Permissions

To restrict page access, you must configure permissions for individual pages within an Appsmith app. For example, to provide access to two different sets of users, you have to create two custom roles. Let's name them **Manage Roles** and **Manage Reports**, and assign appropriate permissions to these roles.

The tables below show the permissions assigned to these roles so that each role has access to different pages in the application. The **Manage Loans** role has access to review loan applications, and the **Manage Reports** role has access to review reports:
