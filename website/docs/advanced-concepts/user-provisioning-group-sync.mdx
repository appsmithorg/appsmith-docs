# Setup SCIM Provisioning

:::info
SCIM Provisioning is only available in Appsmith's [Enterprise Edition](https://www.appsmith.com/pricing).
:::


This page shows how to set up SCIM on your Appsmith instance to streamline the user lifecycle management based on your SSO provider.


## Prerequisites

- A self-hosted Appsmith instance. See the [installation guides](/getting-started/setup/installation-guides) for deploying Appsmith.
- A SAML SSO provider configured on your Appsmith instance. Currently, SCIM is only available for [Active Directory](/getting-started/setup/instance-configuration/authentication/security-assertion-markup-language-saml/active-directory) and [Okta](/getting-started/setup/instance-configuration/authentication/security-assertion-markup-language-saml/okta)

## Enable SCIM provisioning on Appsmith

1. On your instance, go to **Admin Settings > Access Control** and click **Provisioning**.
2. On the **User Provisioning & Group Sync** page, click **Enable**.
3. Copy the **SCIM API endpoint** URL to add it later in your IdP application.
4. Under **API key to setup SCIM**, click the **Generate API key** button and copy the API key to add it later in your IdP application.


## Configure SSO provider

To complete the setup, go to your IdP application that is connected to Appsmith and follow the steps below:

<Tabs queryString="scim">
<TabItem label="Active Direcotry (SAML)" value="scim-ad-saml">

1. On the Active Directory homepage, click **Enterprise application** on the sidebar and open your application connected to Appsmith.

2. On the application **Overview** page, click **Provisioning** on the sidebar.

3. On the **Provisioning** homepage, click **Provisioning** on the sidebar and follow the steps below:

    a. Select **Automatic** from the **Provision Mode** dropdown.

    b. In the **Tenant URL** field, paste the **SCIM API endpoint** URL copied ([step 3](/advanced-concepts/user-provisioning-group-sync#enable-scim-provisioning-on-appsmith)) from Appsmith.

    c. In the **Secret Token** field, paste the **API key** copied ([step 4](/advanced-concepts/user-provisioning-group-sync#enable-scim-provisioning-on-appsmith)) from Appsmith.

    d. Click the **Test Connection** button to check if the added credentials are authorized to enable provisioning.

    e. Expand the **Mappings** section, click **Provision Azure Active Directory Users**, and scroll down to **Attribute Mappings** to configure attributes for users. Delete all the attributes except the following:

    - **userPrincipalName**
    - **displayName**
    - **Switch([IsSoftDeleted], , "False", "True", "True", "False")**

    f. Click **Save**.

    g. Go back to **Mappings** on the **Provisioning** page, click **Provision Azure Active Directory Groups**, and scroll down to **Attribute Mappings** to configure attributes for groups. Delete all the attributes except the following:

    - **displayName**
    - **members**

    h. Click **Save**.

    i. Go back to the **Provisioning** page, turn the toggle **On** for **Provisioning status** present at the bottom. 

4. Go to **Users and Groups** from the sidebar, and click **Add user/group** to add a user or group to your application. These users or groups get automatically synced and added to your Appsmith instance.

5. If you want to sync a user or group manually, go to the **Provision on Demand** from the sidebar, and enter a user or group name in the **Select a user or group** field. 

</TabItem>

<TabItem label="Okta (SAML)" value="scim-okta-saml">

1. Open your Okta application connected to Appsmith and go to the **General** Tab.
2. In the **General** tab, click **Edit** on **App Settings** and select the **Enable SCIM provisioning** checkbox. Click **Save**.
3. Go to the **Provisioning** tab, click **Edit** on SCIM connection.

    a. Paste the **SCIM API endpoint** URL copied ([step 3](/advanced-concepts/user-provisioning-group-sync#enable-scim-provisioning-on-appsmith)) from Appsmith to the **SCIM connector base URL** field.

    b. In **Unique identifier field for users**, enter `userName`.

    c. In **Supported provisioning actions**, select the **Push New Users**, **Push Profile Updates**, and **Push Groups** checkboxes.

    d. Select **HTTP Header** the **Authentication Mode** dropdown.

    e. In the **HTTP Header** section, paste the **API key** copied ([step 4](/advanced-concepts/user-provisioning-group-sync#enable-scim-provisioning-on-appsmith)) from Appsmith to the **Authorization** field.
    
    f. Click **Test Connector Configuration** to test the connection.

    g. Click **Save**.

4. In the Provisioning tab, go to **to App**.

    a. Click **Edit** on the **Provisioning to App** section, and tick all the options. Click **Save**.

    b. Scroll down to **SCIM_test Attribute Mappings**, remove all the attributes except the following:

    - **Username**
    - **Display name**

5. To assign the application to users or groups, go to the **Assignments** tab, click the **Assign** dropdown and select **Assign to People** or **Assign to Group** as per requirement.  

</TabItem>    
</Tabs>

Once the setup is complete, refresh Appsmith in your browser, and the users and groups from your IdP provider sync in Appsmith. You can check the users and groups in Appsmith under **Admin Settings > Access Control > Users/Groups**.
