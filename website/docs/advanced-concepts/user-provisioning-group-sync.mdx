# Setup SCIM Provisioning

User Provisioning and Group Sync uses SCIM, a standardized protocol that simplifies user provisioning and management. On your Appmith instance, It enables you to streamline the user lifecycle management based on your SSO provider and ensure data security effectively. This page describes how to setup SCIM on your Appsmith instance.

## Prerequisites

-  Ensure that you have an SSO provider configured on your Appsmith instance.


## Set up provisioning on Appsmith

1. On your instance, go to **Admin Settings > Access Control** and click **Provisioning**.
2. On the **User Provisioning & Group Sync** page, click **Enable**.
3. Copy the **SCIM API endpoint** URL to add it later in your IdP application.
4. Under **API key to setup SCIM**, click the **Generate API key button and copy the API key to add it later in your IdP applicaton.


## Configure SSO provider

To complete the setup, go to your IdP application that is connect to Appsmith and follow the steps below

1. On the Active Directory homepage, click **Enterprise application** from the sidebar and open your application connect to Appsmith.

2. On the application **Overview** page, click **Provisioning** from the sidebar.

3. On the Provisioning homepage, click **Provisioning** from the sidebar and follow the steps below:

    a. Select **Automatic** from the **Provision Mode** dropdown.

    b. In the **Tenant URL** field, enter the **SCIM API endpoint** URL copied from Appsmith.

    c. In the **Secret Token** field, enter the **API key to setup SCIM** copied from Appsmith.

    d. Under Mappings, click **Provision Azure Active Directory Users** and scroll down to **Attribute Mappings**. Delete all the attributes except the following:

    - **userPrincipalName**
    - **displayName**

    e. Click **Save**

    f. Go back to the **Provisioning** page, turn the toggle **On** for **Provisioning status** present at the bottom.

<TabItem label="Okta (SAML)" value="scim-okta-saml">

1. Open your Okta application connected to Appsmith and go to the **General** Tab.
2. In the **General** tab, click **Edit** on **App Settings** and select the **Enable SCIM provisioning** checkbox. Click **Save**.
3. Go to the **Provisioning** tab, click **Edit** on SCIM connection.

    a. Add the **SCIM API endpoint** URL copied from Appsmith to the **SCIM connector base URL** field.

    b. In **Unique identifier field for users**, enter `userName`.

    c. In **Supported provisioning actions**, select the following checkboxes:

    - **Push New Users**
    - **Push Profile Updates**
    - **Push Groups**

    d. Select **HTTP Header** the **Authentication Mode** dropdown.

    e. In the **HTTP Header** section, add the **API key to setup SCIM** key copied from Appsmith to the **Authorization** field.
    
    f. Click **Test Connector Configuration** to test the connection.

    g. Click **Save**.

4. In the Provisioning tab, go to **to App**.

    a. Click **Edit** on the **Provisioning to App** section, and tick all the options. Click **Save**.

    b. Scroll down to **SCIM_test Attribute Mappings**, remove all the attributes except the following:

    - **Username**
    - **Display name**
