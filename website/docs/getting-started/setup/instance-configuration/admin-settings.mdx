# Admin Settings

Appsmith allows instance administrators to manage settings through the **Admin Settings** page. These settings control all instance-wide configurations, including email settings, authentication methods, session management, and more. This page provides details about these settings and their configurations.

:::caution important
 **Note:** Settings configured via environment variables override any settings configured via the Admin Settings UI.
:::

## General

### Instance name

<dd>
Modify the name of your self-hosted Appsmith instance. This name appears across the platform, including the top bar and login screen.
</dd>

### Admin email

<dd>
Specify the email addresses of users who should have the **Instance Administrator** role. Separate multiple emails with commas.
</dd>

### Share anonymous usage data

<dd>
Opt in or out of sharing anonymous usage data with Appsmith. Shared data helps improve the platform by offering insights into performance and usage.
</dd>

### Appsmith watermark

<dd>
Toggle the display of the **Built on Appsmith** watermark on your applications. To remove this watermark, youâ€™ll need an [Appsmith Commercial Edition](https://www.appsmith.com/pricing) plan.
</dd>

### Programmatic access to users and groups

<dd>
Enable programmatic access to user **roles and groups** within your apps. When enabled, developers can use this data to build complex role-based logic in Appsmith apps.
</dd>

### User session limit

<dd>
Restrict users to a single active session per account. If enabled, logging in from a new device will log out any active sessions.
</dd>

### Session timeout

<dd>
Set the period of inactivity after which users will be automatically logged out. Specify this in `DD:HH:MM` format. The minimum is 1 minute, and the maximum is 30 days.
</dd>

### Atomic pushes to Git

<dd>
Ensures Git operations such as pushes occur atomically, reducing errors during repository synchronization.
</dd>

### Embed settings

<dd>
Control embedding behavior for your apps. Options include:
- **Allow embedding on any domain** - When enabled, Appsmith can be embedded to any domain.
- **Restrict embedding to specific domains** - When enabled, Appsmith can be embedded into the provided list of domains only.
- **Disable embedding entirely** - When enabled, Appsmith cannot be embedded into any domains.
For more information about how to embed Appsmith, see the [Embed Appsmith](/advanced-concepts/embed-appsmith-into-existing-application) guide.
</dd>

## Authentication

<dd>
Appsmith provides different authentication methods through which you can use how users can be authenticated before using Appsmith. Below are some of the available authentication methods that you can configure on Appsmith:
- **OIDC (OpenID Connect)**- Integrate with an OpenID Connect provider. For more information about setting up an OIDC provider, see [OpenID Connect SSO](/getting-started/setup/instance-configuration/authentication/openid-connect-oidc) guides.
- **SAML (Security Assertion Markup Language)**- Enable SAML-based SSO for your organization. For more information about setting up an SAML SSO provider, see [SAML SSO](/getting-started/setup/instance-configuration/authentication/security-assertion-markup-language-saml) guides.
- **Google**- When you want your users to use their Google accounts to access Appsmith, use this authentication method to authenticate users using Google OAuth. For more information about setting up Google OAuth, see [Google OAuth](/getting-started/setup/instance-configuration/authentication/google-login) guide.
- **GitHub**- When you want your users to use their GitHub accounts to access Appsmith, use this authentication method to authenticate users using GitHub OAuth. For more information about setting up GitHub OAuth, see [GitHub OAuth](getting-started/setup/instance-configuration/authentication/github-login) guide.
- **Form Login**- The default username/password authentication method managed by Appsmith.
</dd>

## Branding

### Branding options

<dd>
Customize your Appsmith instance with:

- **Favicon**: Set a favicon to match your brand identity.

## Version

<dd>
Displays the current Appsmith version running on your instance. Use this to track updates and ensure compatibility with new features.
</dd>

## License key

<dd>
Displays the license key for your Appsmith Commercial Edition. Required to unlock advanced features.
</dd>

## Plans and usage

<dd>
View your current subscription plan and its usage details, including user limits and feature availability. For more details, see [Appsmith Pricing](https://www.appsmith.com/pricing).
</dd>

## Access control

### Provisioning

<dd>
Configure how users, groups, and roles are provisioned. Options include manual provisioning or integrating with external identity providers.
</dd>

### Users

<dd>
Manage all users within your Appsmith instance. View user details, assign roles, and deactivate accounts as needed.
</dd>

### Groups

<dd>
Organize users into groups for easier permission management. Assign specific roles or access levels to groups.
</dd>

### Roles

<dd>
Define and manage roles within your Appsmith instance. Roles determine what permissions a user or group has, such as access to specific apps or administrative controls.
</dd>
