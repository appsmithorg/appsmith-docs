# Instance Management
One of the key features of Appsmith is its instance management capabilities. Appsmith provides a variety of tools for managing instances so that you can:

* [Configure a Maintenance Window for auto-updates to cut the downtime for your users](/getting-started/setup/instance-management/maintenance-window)
* [Configure auto-updates to keep your Appsmith installation up-to-date](#auto-updates)
* [Backup and Restore your Appsmith instance using command line utility](/getting-started/setup/instance-management/appsmithctl)

## Auto updates 
Auto update feature ensures that your Appsmith instance is always up-to-date with the latest features and bug fixes. 

Follow the below steps for your Appsmith installation:

<Tabs queryString="current-installation">
  <TabItem label="Docker" value="Docker"> 

1. Go to the root directory of the Appsmith installation and run:

   ```bash
   docker-compose down
   ```

2. Open the `docker-compose.yml` file and comment out the below code block:
   
   ```yaml
   #auto_update:
   #  image: containrrr/watchtower
   #  volumes:
   #    - /var/run/docker.sock:/var/run/docker.sock
   #  # Update check interval in seconds.
   #  command: --schedule "0 0 * ? * *" --label-enable --cleanup
   #  restart: unless-stopped
   #  depends_on:
   #    - appsmith
   #  environment:
   #    - WATCHTOWER_LIFECYCLE_HOOKS=true
   ```

3. Save the file and run:

```bash
docker-compose up -d
```

4. Verify that auto update is turned off with:

```bash
docker-compose ps
```

  </TabItem>

  <TabItem label="Kubernetes" value="Kubernetes">

<Tabs groupId="appsmithEditions" queryString="current-edition">
<TabItem label="Community Edition" value="communityEdition"> 

  You can switch off auto update by either using a command or setting it in the `values.yaml` file.

  * Running the command below.

  ```bash
    #highlight-next-line
    helm upgrade --set autoupdate.enabled=false \
     appsmith appsmith/appsmith
  ```

  * **Or** edit `values.yaml`, update the below parameters:
  ```yaml
    autoupdate.enabled: false
    autoupdate.scheduler: 0
  ```
  and run the below command:
  ```bash
    helm upgrade --values values.yaml appsmith appsmith/appsmith
  ```
